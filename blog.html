Discussion of the scope of var, let, and const

JavaScript, along with other modern programming languages, offers a variety of patterns for declaring variables. These patterns include the use of "var", "let", and "const" keywords, each serving a different purpose. This article aims to provide an insight into the nuances and differences encountered when using these variable declaration patterns. For additional context, please refer to the official ECMAScript specifications and MDN documentation for the definitions of each keyword.

The "var" keyword is used to declare a variable that is scoped to the current execution context, and it is unnecessary to initialize it with a value.

The "let" keyword, on the other hand, declares a block-scoped variable, initializing it to an initial value if desired.

About REST API

While the above description may not appear as complex or "professional" as what you may find online, the goal here is for you to understand the basic purpose of REST APIs. An API that adheres to some or all of the six key constraints of REST is considered RESTful. We can interact with servers using HTTP methods, such as Create, Read, Update, and Delete, also known as CRUD operations. But how can we perform these CRUD operations and manipulate data on the server? This is achieved by sending HTTP requests, which is where REST comes into play.
